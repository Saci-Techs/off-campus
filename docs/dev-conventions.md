# Convenções de Desenvolvimento (Frontend)

Este arquivo contém padrões internos utilizados pela equipe de desenvolvimento.

## Estrutura de CSS

- 1 arquivo de estilo por pasta de componente/página
- Cada página deve possuir sua própria pasta de módulos/componentes relacionados

## Organização de Componentes

- Componentes devem ser agrupados por funcionalidade/página
- Evitar componentes soltos na raiz sem contexto

## Observações gerais

- Ajustes de tipografia (ex: Road Rage font-weight 400) devem ser registrados no global.css quando necessário
- Problemas conhecidos de estilo devem ser documentados aqui quando não forem bugs críticos
